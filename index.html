How it works. Each PNG is decoded and encoded frame-by-frame with the browser's hardware H.264 encoder, then muxed into a real .mp4 container. No re-compression of intermediate files, no server upload.
Note on "lossless". Native browser encoding is H.264 4:2:0 — visually lossless at high bitrate, not bit-exact. For mathematically lossless output you'd need x264 -crf 0 (ffmpeg), which the browser can't do. Same-size frames encode cleanest; differing sizes are centre-padded, never scaled.
